Toys API
All Optional Routers:

HTTP
Method		 Route Description Parameters / Request Body
GET /toys Returns a list
of toys
according to
the query
parameters
you use.		 1.page (optional): page number to display
(default: 1). Example-
/toys?page=1
GET /toys/search/:name or information Returns a list
of toys
matching to
the search
query. 		1.the search query to match against toys names or
information.
Example names-
/toys/search/lego
Example information-
/toys/search/321
GET /toys/category/:catname Returns a list
of toys by
category		 1.catname: the name of the category to display toys for.
Example-
/toys/category/polish
GET /toys/prices Returns a
list of toys
filtered by
price range.		 • min (optional): minimum price to filter by (default: 0).
• max (optional): maximum price to filter by (default:
maximum 999999).
Example-
/toys/prices?min=60&max=110
You can also do a price range search on the page-
/toys/prices?page=1&min=60&max=110
GET /toys/single/:id Returns a
single toy
by its ID.		 • id: the ID of the toy to retrieve.
/toys/single/650026395d6616d0afcbe1e0
GET /toys/count Returns how many
toys there are
in total and
how many pages
it is divided
according to
the limit.
Example-
/toys/count
POST /toys Adds a new toys.
Token is required! 		Request Body: Toy data to be added in the
following format-
{
"name":"send" ,
"info": "321",
"category": "polish",
"img_url": " ",
"price": "75",
"userId": "650023b15d6616d0afcbe1da"
}
PUT /toys/:editId Updates an
existing toys.
Token is required! 		• editId: the ID of the toy to update.
• Request Body: Toy data to be
updated in the following format-
{
"name":"send" ,
"info": "321",
"category": "polish",
"img_url": " ",
"price": "70",
"userId": "650023b15d6616d0afcbe1da"
}
DELETE /toys/:delId Deletes a toy
by its ID.
is required! 		1.delId: the ID of the toy to delete.
